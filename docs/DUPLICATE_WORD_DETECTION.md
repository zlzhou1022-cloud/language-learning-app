# 重复单词检测功能

## 功能概述
在学习界面输入单词时，系统会自动检测该单词是否已存在于用户的词汇库中。如果检测到重复，会根据学习模式采取不同的处理方式。

## 功能逻辑

### 检测时机
- 用户在学习界面输入单词并点击"开始学习"按钮时
- 系统会查询数据库，检查该单词是否已存在

### 处理逻辑

#### 1. 钻研模式（Deep Mode）
**行为**: 自动进入继续学习对话模式

**等价操作**: 在单词详情页点击"继续学习"按钮

**特点**:
- 不显示 AI 风格选择弹窗（直接使用学习页面已选择的风格）
- 进入对话界面，AI 会基于已有卡片内容继续对话
- 对话结束后点击"更新卡片"
- **只更新** `learning_notes`（学习要点）和 `mnemonics`（助记方法）
- 其他字段（音标、释义、例句等）保持不变
- 卡片编辑界面会高亮显示新增的内容

#### 2. 效率模式（Efficient Mode）
**行为**: 直接显示卡片编辑页面

**等价操作**: 在单词详情页点击"编辑"按钮

**特点**:
- 跳过对话环节，直接进入卡片编辑界面
- 显示已保存的卡片内容
- 用户可以手动编辑所有字段
- 保存时**更新所有字段**

## 技术实现

### 1. 数据库查询
```typescript
const { data: existingWords, error } = await supabase
  .from('dictionaries')
  .select('*')
  .eq('user_id', user.id)
  .eq('word', word.trim())
  .limit(1);
```

### 2. 状态管理
添加了以下状态来跟踪重复单词：
- `isExistingWord`: 标记是否是重复单词
- `existingWordId`: 保存重复单词的数据库 ID
- `existingCard`: 保存已有的卡片数据

### 3. 模式判断
```typescript
// 钻研模式：进入继续学习对话
if (mode === 'deep') {
  setStage('chat');
}

// 效率模式：直接显示编辑页面
else {
  setGeneratedCard(card);
  setStage('edit');
}
```

### 4. 保存逻辑
```typescript
// 判断是否只更新学习要点和助记方法
const shouldUpdateNotesOnly = 
  (wordId && editMode === 'continue') ||  // 从词汇详情页继续学习
  (isExistingWord && mode === 'deep');    // 钻研模式下的重复单词

if (shouldUpdateNotesOnly) {
  // 只更新 learning_notes 和 mnemonics
} else {
  // 更新所有字段
}
```

## 用户体验流程

### 场景 1: 钻研模式 + 重复单词

1. 用户在学习界面选择"钻研模式"
2. 选择 AI 风格（学术/日常/速记）
3. 输入已存在的单词，如 "test"
4. 点击"开始学习"
5. **系统检测到重复** → 自动加载已有卡片
6. 直接进入对话界面（使用已选择的 AI 风格）
7. AI 基于已有卡片内容继续对话
8. 用户提问、学习
9. 点击"更新卡片"
10. 进入卡片编辑界面，**新增内容高亮显示**
11. 点击"保存卡片"
12. **只更新** `learning_notes` 和 `mnemonics`
13. 返回词汇页面

### 场景 2: 效率模式 + 重复单词

1. 用户在学习界面选择"效率模式"
2. 输入已存在的单词，如 "test"
3. 点击"开始学习"
4. **系统检测到重复** → 自动加载已有卡片
5. 直接显示卡片编辑界面
6. 用户可以手动编辑所有字段
7. 点击"保存卡片"
8. **更新所有字段**
9. 返回词汇页面

### 场景 3: 新单词（不重复）

1. 用户输入不存在的单词
2. 系统检测到是新单词
3. 按正常流程处理：
   - 钻研模式：进入对话
   - 效率模式：生成卡片

## 与词汇详情页的对比

| 操作 | 学习界面（重复检测） | 词汇详情页 |
|------|---------------------|-----------|
| 钻研模式 + 重复单词 | 不显示风格选择弹窗，使用已选风格 | 显示风格选择弹窗 |
| 效率模式 + 重复单词 | 直接显示编辑页面 | 点击"编辑"按钮 |
| 保存逻辑 | 钻研模式只更新 notes/mnemonics | 继续学习只更新 notes/mnemonics |
| 高亮显示 | 钻研模式显示新增内容高亮 | 继续学习显示新增内容高亮 |

## 优势

1. **无缝体验**: 用户不需要手动检查单词是否已存在
2. **智能处理**: 根据学习模式自动选择最合适的处理方式
3. **避免重复**: 防止用户重复创建相同的单词卡片
4. **保留历史**: 钻研模式下保留原有内容，只追加新的学习要点
5. **灵活编辑**: 效率模式下允许完全编辑已有卡片

## 错误处理

- 如果数据库查询失败，系统会记录错误但继续正常流程（创建新卡片）
- 如果用户未登录，会提示"请先登录"
- 如果保存失败，会显示错误信息

## 状态清理

为了避免状态污染，在返回学习首页时会清空所有相关状态：
```typescript
const handleBackToInput = () => {
  setStage('input');
  setWord('');
  setGeneratedCard(null);
  setExistingCard(null);      // 清空已有卡片
  setIsExistingWord(false);   // 重置重复标记
  setExistingWordId(null);    // 清空单词 ID
  setSavedMessages([]);       // 清空保存的对话
  setSavedProgress(0);
};
```

这确保了：
- 每次学习新单词时都是全新的状态
- 不会出现上一个单词的数据影响下一个单词
- 避免重复检测逻辑错误触发
