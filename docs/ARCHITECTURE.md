## 数据流

### 认证流程

```
用户输入邮箱
    ↓
LoginForm (Client Component)
    ↓
Supabase Auth API
    ↓
发送 Magic Link 邮件
    ↓
用户点击链接
    ↓
/auth/callback (Route Handler)
    ↓
exchangeCodeForSession
    ↓
设置 Cookie
    ↓
重定向到 /dashboard
    ↓
Middleware 验证 Session
    ↓
Dashboard 页面加载
```

### 数据查询流程

```
Dashboard 页面 (Server Component)
    ↓
createClient (Server)
    ↓
supabase.from('dictionaries').select()
    ↓
RLS 策略检查
    ↓
PostgreSQL 查询
    ↓
返回数据
    ↓
渲染页面
```

### 国际化流程

```
用户访问 /
    ↓
Middleware 检测语言
    ↓
重定向到 /[locale]
    ↓
加载对应的 messages/[locale].json
    ↓
NextIntlClientProvider
    ↓
组件使用 useTranslations()
    ↓
显示翻译文本
```

## 设计模式

### 1. 组合模式 (Composition)
```typescript
// 组件组合而非继承
<Card>
  <CardHeader>
    <CardTitle>Title</CardTitle>
  </CardHeader>
  <CardContent>Content</CardContent>
</Card>
```

### 2. 提供者模式 (Provider)
```typescript
// 主题和国际化通过 Provider 注入
<ThemeProvider>
  <NextIntlClientProvider>
    {children}
  </NextIntlClientProvider>
</ThemeProvider>
```

### 3. 工厂模式 (Factory)
```typescript
// Supabase 客户端工厂
createClient() // 根据环境返回不同客户端
```

### 4. 中间件模式 (Middleware)
```typescript
// 请求拦截和处理
middleware.ts → 认证检查 → 国际化 → 路由
```
