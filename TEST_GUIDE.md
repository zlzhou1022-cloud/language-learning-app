## 🚀 快速测试步骤

### 1. 确保开发服务器正在运行

```bash
npm run dev
```

应该看到：
```
✓ Ready in XXXms
- Local: http://localhost:3000
```

### 2. 测试登录页面

打开浏览器，访问以下任一 URL：

**选项 A: 自动重定向（推荐）**
```
http://localhost:3000/
```
→ 自动重定向到 `http://localhost:3000/en/login`

**选项 B: 直接访问**
```
http://localhost:3000/en/login  (英文)
http://localhost:3000/zh/login  (中文)
http://localhost:3000/ja/login  (日文)
```

### 3. 验证登录页面

你应该看到：

✅ 页面标题: "Language Learning"  
✅ 描述文字: "Enter your email to receive a magic link for login"  
✅ 邮箱输入框  
✅ "Send Magic Link" 按钮  

### 4. 测试登录流程

1. **输入邮箱**
   ```
   your-email@example.com
   ```

2. **点击 "Send Magic Link"**

3. **查看结果**
   - 应该显示成功消息: "Check your email for the login link"
   - 检查你的邮箱（包括垃圾邮件文件夹）

4. **点击邮件中的链接**
   - 应该自动登录
   - 重定向到 Dashboard

### 5. 测试 Dashboard

登录后应该看到：

✅ 左侧边栏（桌面）或底部导航（移动）  
✅ 欢迎消息: "Welcome, [your-email]"  
✅ 总词汇量卡片: "Total Words: 0"  
✅ 提示消息: "No words yet. Start adding vocabulary!"  

### 6. 测试语言切换

1. **在侧边栏找到语言切换器**
   - 桌面: 左侧边栏底部
   - 移动: 设置页面

2. **点击语言按钮**
   - 应该看到下拉菜单: English, 中文, 日本語

3. **选择不同语言**
   - URL 应该改变: `/en/` → `/zh/` → `/ja/`
   - 页面内容应该翻译

### 7. 测试主题切换

1. **找到主题切换按钮**
   - 桌面: 左侧边栏底部
   - 移动: 设置页面

2. **点击切换**
   - 应该在明亮/暗黑模式之间切换
   - 颜色应该立即改变

### 8. 测试响应式布局

1. **桌面视图（> 1024px）**
   - 应该看到左侧边栏
   - 导航项垂直排列

2. **移动视图（< 1024px）**
   - 侧边栏应该隐藏
   - 应该看到底部导航栏
   - 导航项水平排列

3. **调整浏览器窗口大小**
   - 布局应该平滑切换

### 9. 测试导航

点击以下导航项，确保页面正常加载：

- ✅ Dashboard
- ✅ Vocabulary（占位页面）
- ✅ Practice（占位页面）
- ✅ Settings（占位页面）

### 10. 测试登出

1. **点击侧边栏底部的 "Logout" 按钮**

2. **验证**
   - 应该重定向到登录页面
   - 尝试访问 `/dashboard` 应该被重定向到 `/login`

## 🎨 设计验证

### 瑞士平面设计风格检查

✅ **无圆角**
- 检查所有按钮、卡片、输入框
- 应该都是直角（0 border-radius）

✅ **大留白**
- 页面应该有充足的空白空间
- 不应该感觉拥挤

✅ **极简配色**
- 明亮模式: 白色背景 + 黑色文字
- 暗黑模式: 深灰背景 + 白色文字
- 无彩色渐变

✅ **清晰层级**
- 标题、正文、次要文字应该有明显区分
- 使用字体大小和颜色深浅区分

✅ **非衬线字体**
- 所有文字应该使用 Inter 字体
- 字体应该清晰易读

## 🐛 常见问题

### 问题: 页面显示 404

**解决**: 确保访问的是带 locale 的路径
```
❌ http://localhost:3000/login
✅ http://localhost:3000/en/login
```

### 问题: 样式不正确

**解决**: 清除缓存并重启
```bash
rm -rf .next
npm run dev
```

### 问题: 登录邮件没收到

**检查**:
1. 邮箱地址是否正确
2. 检查垃圾邮件文件夹
3. 查看 Supabase Dashboard 的日志
4. 确认 Supabase 邮件服务正常

### 问题: 主题切换不工作

**解决**: 
1. 检查浏览器是否支持 localStorage
2. 清除浏览器缓存
3. 重新加载页面

### 问题: 语言切换后内容没变

**解决**:
1. 检查 URL 是否改变
2. 刷新页面
3. 检查浏览器控制台错误

## 📊 性能检查

### 加载速度

1. **打开浏览器开发者工具**（F12）

2. **切换到 Network 标签**

3. **刷新页面**

4. **检查指标**
   - 首次内容绘制（FCP）: < 1s
   - 可交互时间（TTI）: < 2s
   - 总加载时间: < 3s

### Lighthouse 测试

1. **打开 Chrome DevTools**

2. **切换到 Lighthouse 标签**

3. **运行测试**

4. **期望分数**
   - Performance: > 90
   - Accessibility: > 90
   - Best Practices: > 90
   - SEO: > 90
