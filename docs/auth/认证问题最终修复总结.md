## 🐛 问题 1: Magic Link 重复使用漏洞

### 问题描述
用户点击新的 Magic Link 登录后，再点击旧的 Magic Link 仍然可以登录。

### 根本原因
代码先检查用户是否已登录，如果已登录直接跳转，不验证 token 有效性。

### 修复方案
✅ **始终先验证 token，再检查登录状态**

**修改文件**: `app/[locale]/auth/callback/route.ts`

```typescript
// ✅ 修复后的逻辑
const supabase = await createClient();

// 1. 如果有 token，必须先验证
if (tokenHash && type) {
  const { data, error } = await supabase.auth.verifyOtp({
    token_hash: tokenHash,
    type: type as 'magiclink' | 'email',
  });
  
  if (error) {
    // 如果 token 无效但用户已登录，强制登出
    const { data: { user: existingUser } } = await supabase.auth.getUser();
    if (existingUser) {
      await supabase.auth.signOut(); // ✅ 关键修复
    }
    return buildRedirect(request, `${origin}/${locale}/login?message=link_expired`);
  }
  
  if (data?.user) {
    return buildRedirect(request, `${origin}${next}`);
  }
}

// 2. 如果没有认证参数，才检查是否已登录
const { data: { user: existingUser } } = await supabase.auth.getUser();
if (existingUser) {
  return buildRedirect(request, `${origin}${next}`);
}
```

### 安全改进
- ✅ Token 验证优先
- ✅ 一次性使用
- ✅ 强制登出
- ✅ 明确错误提示

## 🐛 问题 2: 错误信息不会清除

### 问题描述
在显示 "链接已过期" 的页面上重新发送邮件后，错误信息仍然显示。

### 根本原因
错误信息是服务端渲染的（SSR），客户端清除 URL 参数不会触发页面重新渲染。

### 第一次修复（失败）
❌ 只在客户端清除 URL 参数，但服务端渲染的 HTML 不会改变

### 最终修复方案
✅ **将错误信息从服务端渲染改为客户端状态管理**

#### 修改 1: 服务端页面组件

**文件**: `app/[locale]/login/page.tsx`

```typescript
// ✅ 修复后 - 将 message 传递给客户端组件
export default async function LoginPage({ searchParams }) {
  const { message } = await searchParams;

  return (
    <div>
      {/* ✅ 传递给客户端组件处理 */}
      <LoginForm initialMessage={message} />
    </div>
  );
}
```

#### 修改 2: 客户端组件

**文件**: `components/auth/login-form.tsx`

```typescript
// ✅ 添加状态管理
export function LoginForm({ initialMessage }: { initialMessage?: string }) {
  const [showUrlError, setShowUrlError] = useState(false);

  // ✅ 初始化错误显示
  useEffect(() => {
    if (initialMessage === 'link_expired') {
      setShowUrlError(true);
    }
  }, [initialMessage]);

  const handleMagicLink = async (e: React.FormEvent) => {
    e.preventDefault();
    setShowUrlError(false); // ✅ 隐藏错误
    clearUrlMessage();      // ✅ 清除 URL
    // ...
  };

  return (
    <div>
      {/* ✅ 客户端控制的错误信息 */}
      {showUrlError && (
        <div className="border-l-2 border-destructive pl-4">
          <p className="text-sm text-destructive">{t('linkExpired')}</p>
        </div>
      )}
      {/* 表单内容 */}
    </div>
  );
}
```

- ✅ 错误信息自动清除
- ✅ 不需要刷新页面
- ✅ 即时反馈
- ✅ 流畅的单页应用体验
