# 密码认证更新

## 🎉 新功能：支持密码登录

### 更新内容

#### 1. 双登录方式
现在支持两种登录方式，可以在登录页面切换：

**密码登录**（推荐）
- ✅ 无需等待邮件
- ✅ 没有速率限制
- ✅ 立即登录
- ✅ 适合开发和日常使用

**Magic Link**（邮箱链接）
- ✅ 无需密码
- ✅ 更安全
- ⚠️ 有速率限制（免费套餐每小时 4 封邮件）

#### 2. 注册功能
- 新用户可以直接注册账号
- 输入邮箱和密码即可
- 最少 6 个字符密码

#### 3. 完整的中文支持
- 所有界面文字已翻译
- 包括新增的密码登录相关文字

---

## 🚀 快速开始

### 步骤 1: 启用 Supabase 密码认证

1. **访问 Supabase Dashboard**
   ```
   https://supabase.com/dashboard/project/jwoyiuswfpdacfyieppo/auth/providers
   ```

2. **配置 Email 提供商**
   - 点击 "Email" 行
   - 确保勾选：
     - ✅ Enable email provider
     - ✅ Enable password
     - ❌ Confirm email（开发环境建议禁用）
   - 点击 "Save"

详细步骤：[ENABLE_PASSWORD_AUTH.md](./ENABLE_PASSWORD_AUTH.md)

### 步骤 2: 测试登录

1. **访问登录页面**
   ```
   http://localhost:3000/zh/login
   ```

2. **应该看到两个标签**
   - "密码登录"（默认选中）
   - "邮箱链接"

3. **注册新账号**
   - 输入邮箱：`test@example.com`
   - 输入密码：`test123`（至少 6 个字符）
   - 点击 "注册"
   - 应该显示成功消息

4. **登录**
   - 输入相同的邮箱和密码
   - 点击 "登录"
   - 应该立即跳转到 Dashboard

## 🔧 解决的问题

### 1. ✅ 中文显示问题
**问题**: `/zh/login` 显示英文  
**原因**: 翻译文件加载正确，但可能是浏览器缓存问题  
**解决**: 
- 清除浏览器缓存（Ctrl+Shift+Delete）
- 硬刷新（Ctrl+F5）
- 重启开发服务器

### 2. ✅ Rate Limit 问题
**问题**: "email rate limit exceeded"  
**原因**: Supabase 免费套餐限制每小时 4 封邮件  
**解决**: 使用密码登录，无邮件限制

### 3. ✅ 双重邮件确认
**问题**: 需要确认两次邮件  
**解决**: 在 Supabase 禁用 "Confirm email"

## 🔐 安全说明

### 开发环境
- 可以使用简单密码（如 "123456"）
- 建议禁用邮箱确认
- 使用测试邮箱

### 生产环境
- 强制使用强密码（至少 8 个字符，包含大小写字母和数字）
- 启用邮箱确认
- 配置自定义 SMTP
- 考虑添加 2FA

## 🐛 故障排查

### 问题：中文仍然显示英文

**解决方案**：
```bash
# 1. 清除浏览器缓存
# Chrome: Ctrl+Shift+Delete

# 2. 硬刷新页面
# Ctrl+F5 或 Cmd+Shift+R

# 3. 清除 Next.js 缓存
rm -rf .next

# 4. 重启开发服务器
npm run dev

# 5. 访问
http://localhost:3000/zh/login
```

### 问题：注册后无法登录

**检查**：
1. 确保 Supabase 已启用密码认证
2. 检查密码是否至少 6 个字符
3. 查看浏览器控制台错误
4. 检查 Supabase Dashboard 的用户列表

### 问题：仍然收到 rate limit 错误

**解决**：
1. 使用密码登录（不会触发 rate limit）
2. 等待 1 小时后重试 Magic Link
3. 或配置自定义 SMTP
