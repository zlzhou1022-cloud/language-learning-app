## 🧪 测试 1: Magic Link 一次性使用

### 目标
验证旧的 Magic Link 不能重复使用

### 步骤

#### 1.1 发送第一封 Magic Link
```
1. 访问: http://localhost:3000/en/login
2. 输入你的邮箱地址
3. 点击 "Send Magic Link"
4. 等待成功提示: "Please check your email"
```

**预期结果**: ✅ 显示成功消息

#### 1.2 点击第一封邮件的链接
```
1. 打开邮箱
2. 找到标题为 "Sign in to Language Learning" 的邮件
3. 点击邮件中的 "Sign In" 按钮或链接
```

**预期结果**: 
- ✅ 自动跳转到 `http://localhost:3000/en/dashboard`
- ✅ 看到仪表板页面
- ✅ 控制台显示: `[auth/callback] verifyOtp success`

#### 1.3 发送第二封 Magic Link（可选）
```
1. 登出（如果有登出按钮）或打开隐身窗口
2. 再次访问登录页
3. 输入同一邮箱
4. 点击 "Send Magic Link"
```

**预期结果**: ✅ 收到第二封邮件

#### 1.4 点击第二封邮件的链接
```
1. 打开第二封邮件
2. 点击链接
```

**预期结果**: 
- ✅ 登录成功
- ✅ 跳转到 dashboard

#### 1.5 🔑 关键测试：点击第一封邮件的链接
```
1. 保持登录状态
2. 回到邮箱
3. 找到第一封邮件
4. 再次点击第一封邮件中的链接
```

**预期结果**: 
- ✅ **被登出**
- ✅ 重定向到登录页
- ✅ 显示错误: "链接已过期" 或 "Email link is invalid or has expired"
- ✅ 控制台显示:
  ```
  [auth/callback] verifyOtp error: Email link is invalid or has expired
  [auth/callback] logging out user due to invalid token
  ```

**❌ 如果失败**: 
- 如果仍然能登录，说明修复未生效
- 检查 `app/[locale]/auth/callback/route.ts` 是否正确更新

## 🧪 测试 2: 错误信息自动清除

### 目标
验证 "链接已过期" 错误信息会在重新发送邮件后自动清除

### 步骤

#### 2.1 触发错误信息
```
1. 点击一个过期的 Magic Link（使用测试 1 中的旧链接）
   或者直接访问: http://localhost:3000/en/login?message=link_expired
```

**预期结果**: 
- ✅ 看到红色错误提示: "链接已过期"
- ✅ URL 包含 `?message=link_expired`

#### 2.2 🔑 关键测试：重新发送邮件
```
1. 在显示错误的页面上
2. 输入邮箱地址
3. 点击 "Send Magic Link"
```

**预期结果**: 
- ✅ 错误信息**立即消失**
- ✅ 显示成功消息: "Please check your email"
- ✅ URL 变为: `http://localhost:3000/en/login` (无参数)
- ✅ 控制台无错误

**❌ 如果失败**: 
- 如果错误信息仍然显示，说明 URL 参数未清除
- 检查 `components/auth/login-form.tsx` 中的 `clearUrlMessage()` 调用

#### 2.3 测试切换登录模式
```
1. 再次访问: http://localhost:3000/en/login?message=link_expired
2. 点击 "Password Login" 标签
```

**预期结果**: 
- ✅ 错误信息消失
- ✅ URL 变为: `http://localhost:3000/en/login` (无参数)
- ✅ 显示密码登录表单

#### 2.4 测试密码登录时清除错误
```
1. 再次访问: http://localhost:3000/en/login?message=link_expired
2. 切换到 "Password Login"
3. 输入邮箱和密码
4. 点击 "Login"
```

**预期结果**: 
- ✅ 错误信息消失
- ✅ URL 参数被清除
- ✅ 显示登录结果（成功或失败）

## 🐛 常见问题

### Q1: 收不到邮件
**A**: 
- 检查 `.env.local` 中的 Resend API Key
- 检查邮箱的垃圾邮件文件夹
- 查看服务器控制台是否有错误

### Q2: 点击旧链接仍然能登录
**A**: 
- 确认 `app/[locale]/auth/callback/route.ts` 已更新
- 重启开发服务器: `Ctrl+C` 然后 `npm run dev`
- 清除浏览器缓存和 cookies

### Q3: 错误信息不消失
**A**: 
- 确认 `components/auth/login-form.tsx` 已更新
- 检查浏览器控制台是否有 JavaScript 错误
- 尝试硬刷新: `Ctrl+Shift+R`

### Q4: 控制台看不到日志
**A**: 
- 日志在**服务器端**，查看运行 `npm run dev` 的终端
- 不是浏览器的控制台
