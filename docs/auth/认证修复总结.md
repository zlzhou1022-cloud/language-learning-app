## 🐛 修复的问题

### 问题 1: 旧的 Magic Link 可以重复使用 ⚠️ 高风险

**原因**: 
- 代码先检查用户是否已登录
- 如果已登录，直接跳转，不验证 token
- 导致旧链接可以绕过验证

**修复**: 
- ✅ 调整逻辑顺序：**先验证 token，再检查登录状态**
- ✅ 如果 token 无效但用户已登录，**强制登出**
- ✅ 确保 Magic Link 真正的一次性使用

**影响文件**: `app/[locale]/auth/callback/route.ts`

---

### 问题 2: 错误信息不会自动清除 ❌ 用户体验差

**原因**: 
- URL 参数 `?message=link_expired` 持久化
- 服务端渲染的错误信息不会自动更新
- 用户重新发送邮件后仍看到错误

**修复**: 
- ✅ 在表单提交时调用 `clearUrlMessage()`
- ✅ 在切换登录模式时清除 URL 参数
- ✅ 使用 `window.history.replaceState()` 更新 URL

**影响文件**: `components/auth/login-form.tsx`

---

## 📝 修改的代码

### 文件 1: `app/[locale]/auth/callback/route.ts`

```typescript
// ✅ 修复后的逻辑顺序
const supabase = await createClient();

// 1. 如果有 token，必须先验证
if (tokenHash && type) {
  const { data, error } = await supabase.auth.verifyOtp({
    token_hash: tokenHash,
    type: type as 'magiclink' | 'email',
  });
  
  if (error) {
    // 如果 token 无效但用户已登录，先登出
    const { data: { user: existingUser } } = await supabase.auth.getUser();
    if (existingUser) {
      await supabase.auth.signOut(); // ✅ 强制登出
    }
    return buildRedirect(request, `${origin}/${locale}/login?message=link_expired`);
  }
  
  if (data?.user) {
    return buildRedirect(request, `${origin}${next}`);
  }
}

// 2. 如果没有认证参数，才检查是否已登录
const { data: { user: existingUser } } = await supabase.auth.getUser();
if (existingUser) {
  return buildRedirect(request, `${origin}${next}`);
}
```
